## 📁 What goes in your repository

Your repo should contain **exactly these files and folders**:

```
your-repo/
├── index.html
├── styles.css
├── script.js
└── assets/
    ├── pants.svg
    ├── shorts.svg
    └── jacket.svg
```

That's it. No build tools, no npm, no config files needed.

## 🚀 How to deploy on GitHub Pages

### Step 1 — Create a new GitHub repository

1. Go to **github.com** → click the **+** icon → **New repository**
2. Name it anything, e.g. `czed` or `czed-website`
3. Set to **Public** (required for free GitHub Pages)
4. **Do NOT** check "Add a README file"
5. Click **Create repository**

### Step 2 — Upload your files

On the empty repo page, click **"uploading an existing file"**:

1. Drag and drop ALL of these at once:
   - `index.html`
   - `styles.css`
   - `script.js`
   - The entire `assets/` folder

2. Write a commit message: `Launch C'ZED website`
3. Click **Commit changes**

### Step 3 — Enable GitHub Pages

1. Go to your repo → click **Settings**
2. Left sidebar → click **Pages**
3. Under Source → **Deploy from a branch**
4. Branch: **main** | Folder: **/ (root)**
5. Click **Save**

### Step 4 — Your site is live

Wait 1–2 minutes, then visit:
```
https://YOUR_USERNAME.github.io/YOUR_REPO_NAME/
```
